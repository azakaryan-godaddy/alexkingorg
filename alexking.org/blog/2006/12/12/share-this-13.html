Share This 1.3

December 12, 2006

I’ve released version 1.3 of my Share This plugin for WordPress. This version has two main changes:

  1. Support for a Share This link in feeds via a custom page that presents the Share This information along with the post content. This also works for mobile visitors and visitors that have JavaScript disabled in their browser.
  2. The addition of the Share Icon to the link – on by default.

Here is a screenshot of the Share This custom page:

This screenshot has some of my own custom styling that I applied through the ‘akst_head’ hook (which I added to allow you to customize/override the CSS of the Share This page via plugin).

The icon is, of course, the icon from the Share Icon project (which I launched the web site for yesterday).

I’d also like to thank the folks that sent in a donation after my little plea when I released my Articles plugin. When I receive a few donations like this, it makes it much easier to spend the time necessary to package up these plugins and make them freely available. Any little amount is appreciated – perhaps I should try resurrecting the $1 donation for each download concept again. I still think that’s an idea that could work. If even a fraction of folks who download would do it, it would work out well for everyone.

The download is available on my WordPress Plugins page.

If you have any trouble with this, please open a thread in the WP Support Forums and send me the link.

This post is part of the following projects: Share Icon, ShareThis. View the project timelines for more context on this post.

  21. Alex
    View January 1, 2007

    I think there may be a little FUD here. In a standard WordPress install, your content is already duplicated numerous times:

    1. Permalink
    2. Home page and “paged” history view when it rolls of the history page.
    3. Year archive view
    4. Month archive view
    5. Day archive view
    6. Category archive view (* # of categories you place the post in)

    Anyone who uses a tagging plugin could add an additional n views for the tags on each post.

    In addition, my 1.x mobile edition plugin created alternate URLs for mobile versions of the content and I never heard of any penalties because of it.
